# Command Doc Validator

这个 skill 帮助你编写和验证给 AI Agent 使用的命令运行文档，确保子 Agent 按照文档第一次尝试就能成功执行。

## 核心理念

AI Agent 执行命令失败的主要原因：
- 文档过于冗长，Agent 难以快速定位关键信息
- 命令语法模糊，Agent 需要猜测（如混淆 bash 和 PowerShell 语法）
- 环境差异未明确说明（如 Claude Code 在 Windows 上也用 bash 语法）

解决方法：编写极简、准确的文档，然后用新的子 Agent 测试并统计尝试次数。

## 工作流程

### 1. 理解需求

首先明确：
- 需要执行什么命令？
- 命令的参数有哪些？
- 有哪些常见的执行错误？
- 目标环境是什么？（Claude Code、本地终端等）

### 2. 编写文档草稿

按照以下原则编写：

**极简原则**
- 只保留可执行信息：命令、参数说明、错误解决方案
- 删除理论性内容：流程图、最佳实践、示例输出、执行特点

**准确性优先**
- 命令必须可直接复制粘贴执行
- 明确环境差异（特别重要：Claude Code 统一用 bash 语法，即使在 Windows 上）
- 如果有编码问题、环境变量等，必须在"重要"部分明确说明

**结构清晰**
```markdown
# 标题

## 快速开始
[核心命令，可直接复制执行]

**重要：**
- [关键注意事项，如环境变量、语法差异]

## 参数说明
[简洁表格，3列：参数、默认值、说明]

## 常见问题
[只保留必要的错误和解决方案]

## 环境要求
[简短列出]
```

**关键点：在"重要"部分明确说明环境差异**

例如：
```markdown
**重要：**
- 必须设置 `PYTHONIOENCODING=utf-8`，否则会因 emoji 和中文输出导致编码错误
- 在 Claude Code 中统一使用 bash 语法（即使在 Windows 上也用 `PYTHONIOENCODING=utf-8`，不要用 PowerShell 的 `$env:` 语法）
```

### 3. 用子 Agent 验证

启动一个新的子 Agent（Bash 类型）来测试文档：

**提示词模板：**
```
请阅读 [文档路径]，按照文档运行 [命令]。

最后总结：
1. 尝试了多少次才成功
2. 失败原因是什么
3. 文档是否清晰准确
```

**关键点：**
- ❌ 不要写"确保一次成功"（这是对文档的要求，不是对 Agent 的要求）
- ✅ 让 Agent 自主尝试并统计次数
- ✅ 要求反馈文档问题

使用 Task 工具启动子 Agent：
```
Task(
  description="测试运行文档的准确性，统计尝试次数",
  prompt="请阅读 [文档路径]，按照文档运行 [命令]。\n\n最后总结：\n1. 尝试了多少次才成功\n2. 失败原因是什么\n3. 文档是否清晰准确",
  subagent_type="Bash"
)
```

### 4. 分析反馈并改进

根据子 Agent 的反馈：
- **如果尝试次数 = 1**：文档合格 ✅
- **如果尝试次数 > 1**：分析失败原因，改进文档

常见问题和改进方向：
- **语法错误**：添加明确的环境说明（如"Claude Code 用 bash 语法"）
- **参数不清楚**：补充参数说明或默认值
- **编码错误**：在"重要"部分添加环境变量设置说明
- **文档太长**：删除冗余内容，只保留核心信息

### 5. 迭代直到一次成功

重复步骤 3-4，直到：
- 子 Agent 一次成功
- 零歧义（不需要猜测命令格式）
- 可复现（多次测试都一次成功）

## 成功标准

- **一次成功**：子 Agent 第一次尝试就运行成功
- **零歧义**：子 Agent 不需要猜测或推断命令格式
- **可复现**：多次测试都能一次成功

## 核心经验

1. **文档质量 = 执行成功率**
   - 冗长的文档会分散 Agent 注意力
   - 模糊的说明会导致 Agent 猜测和尝试
   - 明确的约束能避免常见错误

2. **必须验证**
   - 不要假设文档是准确的
   - 使用新的子 Agent 测试（模拟新用户）
   - 统计尝试次数是最直观的质量指标

3. **明确环境差异**
   - Claude Code 的 Bash 工具在 Windows 上也使用 bash 语法
   - 不能假设 Agent 知道这个细节
   - 必须在文档的"重要"部分明确说明

4. **提示词设计**
   - 不要在提示词中要求"一次成功"（这会掩盖文档问题）
   - 让 Agent 自主尝试并反馈
   - 要求统计尝试次数

5. **迭代改进**
   - 第一版文档很难完美
   - 根据子 Agent 的反馈持续改进
   - 目标是让尝试次数降到 1

## 工作示例

**场景：** 用户说"帮我写个 Python 脚本的运行文档，需要设置环境变量和传参数"

**步骤：**

1. **理解需求**
   - 命令：`python script.py --arg1 value1 --arg2 value2`
   - 环境变量：`PYTHONIOENCODING=utf-8`
   - 目标环境：Claude Code

2. **编写文档草稿**
   ```markdown
   # Script 运行指南

   ## 快速开始

   ```bash
   PYTHONIOENCODING=utf-8 python script.py --arg1 value1 --arg2 value2
   ```

   **重要：**
   - 必须设置 `PYTHONIOENCODING=utf-8`
   - Claude Code 统一用 bash 语法（即使 Windows 上也用 `PYTHONIOENCODING=utf-8`，不要用 `$env:`）

   ## 参数说明

   | 参数 | 默认值 | 说明 |
   |------|--------|------|
   | --arg1 | 无 | 参数1说明 |
   | --arg2 | 无 | 参数2说明 |
   ```

3. **启动子 Agent 测试**
   ```
   Task(
     description="测试 Python 脚本运行文档",
     prompt="请阅读运行文档，按照文档运行脚本。最后总结尝试次数、失败原因、文档是否清晰",
     subagent_type="Bash"
   )
   ```

4. **分析反馈**
   - 如果一次成功：文档合格 ✅
   - 如果失败：根据失败原因改进文档，重复步骤 3

## 注意事项

- 不要过度优化：文档是给 AI 看的，不是给人看的，保持简洁即可
- 不要添加过多约束：用"重要"部分说明关键点，而不是用大量 MUST/NEVER
- 关注实际问题：根据子 Agent 的真实反馈改进，不要臆测问题
