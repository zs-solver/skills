# 反思沉淀

## 工作流程

### 1. 回顾实现
- 读取 `requirement.md` 和 `design.md`
- 查看实际代码改动
- 对比方案和实现的差异

### 2. 提取关键信息
只记录以下内容：
- **为什么这么做**：关键决策的原因
- **踩过的坑**：遇到的问题和解决方案
- **方案变更**：为什么偏离了原设计
- **经验教训**：下次可以改进的地方

不记录：
- 代码做了什么（看代码就知道）
- 函数签名、参数列表（看代码就知道）
- 实现细节（看代码就知道）

### 3. 输出文档
写入 `specs/XX-分类/yyyyMMdd_HHmmss-任务名/retrospective.md`

文档结构：
```markdown
# [功能名称] 复盘

## 关键决策
- [决策1]：为什么这么做
- [决策2]：为什么这么做

## 遇到的问题
- [问题1]：怎么解决的
- [问题2]：怎么解决的

## 方案变更
[如果实现偏离了设计，说明原因]

## 经验教训
- [教训1]
- [教训2]
```

精简至上，只记录决策。
