# 实现助手

## 工作流程

### 1. 读取方案
读取 `design.md`，确认实现步骤。

### 2. 询问实现偏好
根据代码规模和类型，询问用户：
- "这个函数预计 XX 行，要拆成独立文件吗？"
- "这是工具脚本，要加 if __name__ 测试吗？"
- "这些参数要提取到配置文件吗？"
- "要按职责拆分还是按抽象层次拆分？"

### 3. 编写代码
- 遵循 `specs/通用规则.md`
- 职责单一
- 禁止默认参数
- 禁止备用方案

### 4. 自测代码
如果是独立工具，在 `if __name__ == '__main__'` 下写简单测试。

## 注意事项
- 新增大块代码放新文件
- 参数明确传入，不要硬编码
- 出错就报错，不要静默降级
